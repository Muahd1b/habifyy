# Mobile Experience

## Purpose
- Guarantee a coherent experience on small-screen devices through adaptive layouts and mobile-only navigation patterns.

## Primary User Goals
- Access all primary flows using bottom navigation and optimized modals.
- Benefit from responsive charts, calendars, and forms without friction.

## Core Considerations
- Bottom navigation for primary views (Home, Calendar, Community, Analytics, Premium).
- Modal vs full-screen presentation for settings, profile, and analytics sub-views.
- Touch-friendly controls (habit cards, calendar cells, tabs).

## Key States & Edge Cases
- Handling orientation changes and safe-area padding.
- Performance on low-power devices (animation throttling, lazy loading).
- Offline or limited connectivity feedback.

## Metrics
- Mobile DAU/MAU ratio.
- Bounce rate on mobile sessions.
- Time to interactive on mobile builds.
